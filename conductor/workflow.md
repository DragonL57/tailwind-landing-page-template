# Workflow

## Development Standards

- **Test Coverage:** >80%
- **Commit Strategy:** Per-task commits
- **Code Style:** Follow project conventions

## Process

1. **Planning:** Define spec before implementation
2. **Implementation:** Write feature with tests
3. **Review:** Self-review before commit
4. **Verification:** Run tests and lint

## Quality Gates

- All tests must pass
- No TypeScript errors
- No linting warnings
